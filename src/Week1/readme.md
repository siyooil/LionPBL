# 1주차 과제

## 과제명
콘솔 기반 아기사자 명단 관리 프로그램 구현 및 객체지향 리팩토링

## 작업내용
* **요구사항 구현**: 
  * `Scanner`를 활용하여 콘솔에서 아기사자 수와 이름을 입력받고 출력하는 기본 흐름 완성
  * 5명 미만 입력 시 오류 메시지 출력 및 프로그램 종료(또는 재입력) 로직 구현
* **코드 리팩토링 (객체지향 설계 적용)**:
  * 전체 로직이 담겨있던 단일 메서드를 역할별로 분리하여 유지보수성 향상
  * **상수화 및 캡슐화**: 반복적으로 사용되는 이모지(`🦁`)와 안내 문구들을 `final static` 상수로 정의하여 코드의 가독성을 높이고, 오타로 인한 버그 방지 및 메모리 효율성 증대
  * 상수 네이밍 컨벤션(`UPPER_SNAKE_CASE`) 적용을 통해 코드의 의미를 명확히 함
  * 필드 변수(상태)와 로컬 변수의 충돌(Shadowing)을 해결하고, 불필요한 `this.` 키워드 제거
* **예외 처리 보완**: 
  * 숫자 입력란에 문자열이 들어왔을 때 발생하는 무한 루프 문제를 `input.hasNextInt()`와 `input.nextLine()`(버퍼 비우기)을 조합하여 안전하게 예외 처리되도록 개선
* **빌드 환경 최적화**: 
  * `gradle.properties` 설정을 통해 Gradle 빌드 속도 개선 (병렬 처리, 캐싱, 메모리 할당)

## 느낀점
* 처음에는 기능이 돌아가게만 만드는 데 집중했지만, 메서드를 분리하고 변수명을 고민하는 과정에서 '객체지향적인 코드 작성'의 중요성을 깨달았습니다.
* 특히 이모지나 메시지를 상수로 관리하니 코드가 훨씬 깔끔해졌고, 나중에 문구를 바꿀 때도 한 곳만 수정하면 된다는 점이 매우 편리했습니다.
* `Scanner`를 사용할 때 버퍼에 남은 찌꺼기(`\n`) 때문에 발생하는 예외 상황을 겪으며, 입력값 검증 로직을 더 견고하게 만들어야 한다는 점을 배웠습니다.
* 코딩 컨벤션(상수 네이밍 등)과 적절한 메서드명(`isRetry`, `isValidLionNumberInput` 등)을 선택하는 것이 다른 사람(또는 미래의 나)이 코드를 읽을 때 얼마나 큰 도움이 되는지 알게 되었습니다.
